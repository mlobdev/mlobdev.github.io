Project STARS (UE5)

STARS is an ongoing 3D Hack and Slash project focused on solving complex gameplay, rendering, and performance challenges in a mixed-gravity planetary environment.

Character Systems & Control Architecture
  • Layered State Machine Architecture: Development of a multi-layered state system separating movement physics, input logic, and animation into independent but coordinated layers. Enables consistent behavior across radically different traversal contexts while maintaining responsiveness and system clarity.
  • Context-Driven Input System: Creation of a flexible input framework where actions dynamically adapt to gameplay context, eliminating hardcoded branching and allowing scalable feature growth.
  • Combo & Combat Prototyping: Iterative development of a responsive combo system focused on precise input timing and fluid chaining, designed to remain stable across variable gravity conditions.
Rendering & Shader Engineering
  • Extreme Shader Optimization: Aggressive use of unlit materials and custom lighting approximations to drastically reduce rendering cost while preserving visual readability.
  • Custom Shadow Approximation: Implementation of shadow behavior in unlit pipelines using vector-based calculations, bypassing standard lighting costs.
  • Advanced Material Systems: Development of triplanar projection workflows and highly customizable materials to accelerate environment creation without UV dependency.
  • Stylized Post-Processing: Creation of custom post-process materials for effects such as cartoon water and underwater rendering, extending the visual pipeline beyond default engine capabilities.
Systems & Engine-Level Design
  • Global Planetary Registry: Implementation of a centralized subsystem providing global access to planetary data, enabling any gameplay system to operate coherently within dynamic gravity fields.
  • Blueprint Micro-Optimization: Continuous refinement of Blueprint systems to minimize overhead and ensure runtime efficiency in complex gameplay scenarios.
  • C++ Driven Architecture: Core systems implemented in C++ and exposed to Blueprint, balancing performance with iteration speed and designer accessibility.
